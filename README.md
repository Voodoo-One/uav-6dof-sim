# UAV 6-DOF Guidance, Navigation, and Control Simulation

A full 6-DOF fixed-wing UAV simulation built from scratch in Python, implementing a complete GNC stack: numerical trim, cascaded PID flight control, cross-track error path following, Dryden atmospheric turbulence, and an Extended Kalman Filter for IMU/GPS navigation. Developed as a portfolio project while finishing a BS in Aerospace Engineering at UCLA, continuing into an MS focused on Systems and Control.

This project originated as an extended response to a class assignment in UCLA MAE 154A (Aerospace Design Laboratory), which is why the aircraft geometry, aerodynamic derivatives, and inertia estimates reflect a specific custom airframe design from that course rather than a generic reference aircraft. Everything beyond the baseline aerodynamic model (the control architecture, guidance algorithms, turbulence model, and EKF) was built independently as an exercise in applying GNC concepts to a real vehicle.

## What it does

The simulation runs a 500-second autonomous waypoint navigation and loiter mission. Starting at 7,000 ft, the UAV climbs through four waypoints reaching 13,000 ft, then enters a 1,500 ft radius loiter orbit; all in a 21 ft/s crosswind with MIL-spec Dryden turbulence. An EKF fuses simulated IMU and GPS data throughout, with its estimated trajectory plotted against ground truth.

---

## Architecture

The project is split into eight modules with clean separation between layers:

| File | Role |
|---|---|
| `UAV_Model.py` | Aerodynamic coefficients, ISA atmosphere, force/moment calculation |
| `UAV_EOM.py` | 6-DOF rigid-body equations of motion, RK4 integration |
| `UAV_Trim.py` | Numerical trim solver (SLSQP) for steady level flight |
| `UAV_Control.py` | Cascaded PID autopilot with back-calculation and conditional anti-windup |
| `UAV_Guidance.py` | Waypoint sequencing, cross-track error path following, tangent loiter law |
| `UAV_Wind.py` | Dryden turbulence model per MIL-HDBK-1797 |
| `UAV_EKF.py` | 10-state Extended Kalman Filter, IMU prediction + GPS correction |
| `UAV_Sim.py` | Top-level simulation loop and plotting |

## Key design decisions

**Quaternion attitude representation.** The EOM and EKF both use quaternions instead of Euler angles to avoid gimbal lock. Attitude is maintained as a unit quaternion throughout; normalization is enforced after every integration step.

**Cascaded PID with altitude-aware trim feedforward.** The autopilot uses separate inner/outer loops for both longitudinal and lateral channels. Trim values are solved numerically at the initial flight altitude using ISA density rather than sea-level defaults — at 7,000–13,000 ft, density is 19–33% below sea level, which introduces a meaningful feedforward error if ignored.

**Two anti-windup strategies.** Inner loops (pitch, roll, speed) use back-calculation anti-windup, appropriate when proportional gain keeps output near the saturation limit. The altitude outer loop uses conditional integration, back-calculation over-unwinds the integrator during multi-thousand-foot climbs because `kp × large error` produces an unsaturated output hundreds of times larger than the limit. The sign correction divides by `ki` to handle negative gains correctly, which standard textbook formulations omit.

**Dryden turbulence via shaping filters.** Rather than Gaussian white noise, turbulence is generated by passing white noise through rational transfer functions that reproduce the Dryden power spectral density (MIL-HDBK-1797 §3.7.2). The longitudinal axis uses a first-order filter; lateral and vertical use second-order filters, consistent with the MIL spec's isotropic high-altitude model. The time constant τ = L/V is recomputed each step so that the spatial scale length stays constant as airspeed changes.

**EKF with analytically-derived Jacobian.** The filter linearises the nonlinear process model around the current state estimate. The only non-trivial Jacobian block is `d(R(q)·f)/dq`, the sensitivity of predicted velocity to attitude. This is derived analytically by differentiating each element of the quaternion rotation matrix and verified against numerical finite differences (error < 1e-7). The Joseph form covariance update is used for numerical stability.

## Aircraft parameters

| Parameter | Value |
|---|---|
| Weight | 320.6 lbf |
| Wingspan | 11.55 ft |
| Wing area | 11.12 ft² |
| Aspect ratio | 12 |
| Cruise speed | 222 ft/s (~152 mph) |
| Max thrust (SL) | 410 lbf |
| CD0 | 0.016 |
| Oswald efficiency | 0.706 |

Aerodynamic coefficients are a mix of legacy wind tunnel data and XFLR5 vortex-lattice results scaled to the actual geometry. Inertia values are estimated by scaling AAI RQ-2 Pioneer values by mass and geometric ratios. A measured bifilar pendulum test would replace these in a real build.

## EKF sensor model

| Sensor | Parameter | Value |
|---|---|---|
| IMU accelerometer | σ | 0.5 ft/s² |
| IMU gyroscope | σ | 0.005 rad/s |
| GPS horizontal | σ | 16.4 ft (~5 m) |
| GPS vertical | σ | 32.8 ft (~10 m) |
| GPS velocity | σ | 0.98 ft/s (~0.3 m/s) |

GPS updates at 1 Hz; IMU prediction runs at 20 Hz. Position estimation RMS error is consistent with GPS horizontal noise (~16 ft), confirming the filter is neither over nor under-confident.

## Running it

```bash
pip install numpy scipy matplotlib pillow
python UAV_Sim.py
```

The simulation prints waypoint capture events as they happen and produces three plots on completion: 3D trajectory (true + EKF estimate), EKF position error over time, and control surface deflections during the final 10 seconds of loiter.

To generate the animated GIF, uncomment the `animate_trajectory` call at the bottom of `UAV_Sim.py`.
